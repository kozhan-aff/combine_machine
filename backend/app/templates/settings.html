зачем это
За один запуск проверки пробьём whois не более этого числа доменов (лучшие по RD — первыми). Остальные подождут следующего прогона.

Кап Ahrefs-проверок за прогон — платная капча за штуку
зачем это
За один запуск проверки сделаем не более этого числа платных Ahrefs-запросов (DR/backlinks) — только для доменов без RD из фида (cctld/reg_ru/ sweb), только для приобретаемых выживших. 0 — полностью выключить Ahrefs- обогащение (authority-балл останется 0, остальной скоринг идёт как обычно).
{{ s.max_ahrefs_per_run }} платных Ahrefs-запросов за прогон максимум (0 = выключено)
Веса критериев — по чему машина судит донора
зачем это
Итоговый скор — взвешенная сумма пяти сигналов. Здесь задаётся, что важнее. Сумма весов может быть любой: доля каждого считается от суммы (справа), так что шкала 0..1 и пороги approve/manual не «плывут» от сдвига одного ползунка. Действует со следующего прогона; уже оценённые домены не пересчитываются — поэтому счётчики у порогов выше на сдвиг весов НЕ реагируют (они считают по сохранённым score). Чтобы веса применились, домен надо оценить заново.
