## MCP — важно
Коннектор **Ahrefs MCP, подключённый в твоём чате Claude, — это для ad-hoc ресёрча
в чате, а не для рантайма пайплайна.** Docker-приложение должно ходить по своим
API-ключам. Не проектируй пайплайн вокруг MCP — он не будет доступен воркеру.
(MCP полезен тебе лично: быстро проверить нишу/домен руками в разговоре.)

## Браузер — нужен ли
Для основного цикла **нет: у каждого шага есть API.** Единственное место соблазна —
скрейпинг выдачи для анализа конкурентов; но **сырой браузерный скрейпинг Google SERP
упирается в CAPTCHA и нарушает ToS** — вместо него SERP-API (DataForSEO/SerpApi).
Итог: **браузер в рантайме не используем.** Браузерная автоматизация оправдана лишь
для разовых ручных задач с сервисами без API — но не в автоматической петле.

## Минимальный набор для MVP
Чтобы пройти петлю на одном домене: метрики-провайдер (1 шт), backorder+optimizator,
Cloudflare, aaPanel, LLM 8833, GSC, Wayback, РКН. SERP/keyword-провайдер и блэклисты
можно подключить на Фазе 2 (улучшение скоринга и контента), но интеграции-стабы уже заложены.
