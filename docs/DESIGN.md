# DESIGN.md — дизайн-система панели КОМБАЙН

Холодная минимальная CMS для конвейера VPN-портфеля. Ориентир — панели управления
(Linear / GitHub / Vercel): плотно, холодно, структура видна с одного взгляда.
Весь визуальный контракт — инлайн в `backend/app/templates/base.html`. Контент-шаблоны
несут только семантику + классы контракта; новые цвета/токены в шаблонах не заводить.

## Токены (`:root` в base.html)

| Группа | Переменные |
|---|---|
| Фон/панели | `--bg #f7f8fa` · `--panel #fff` · `--panel2 #f2f4f7` |
| Линии | `--line #e6e8ec` · `--line2 #d5d9e0` |
| Текст | `--ink #1a1d23` · `--mut #5c626d` · `--dim #9aa1ad` |
| Акцент (система/действие) | `--acc #2563c9` · `--acc2 #1d4ed8` · `--acc-soft #e8f0fe` |
| Статусы | `--ok` зелёный · `--bad` красный · `--warn` жёлтый · `--info` бирюза · `--violet` выкуп |
| Форма | `--r 8px` · `--r-sm 6px` · `--shadow` (одна лёгкая) |

Акцент `--acc` — единственный «активный» цвет: primary-кнопки, активный пункт меню,
ссылки, фокус, гейт-индикаторы. Статусные цвета семантические, не декоративные.

## Типографика

- **UI:** IBM Plex Sans (`--ui`) — инженерный характер, полная кириллица.
- **Данные:** JetBrains Mono (`--mono`) — числа, коды, score/RD, шапки таблиц.
- **Логотип:** Unbounded (`--brand`) — единственная «характерная» деталь.

## Компоненты (классы контракта)

Кнопки `.btn`/`.btn-acc`(primary синий)/`.btn-ok`/`.btn-bad`/`.btn-buy`/`.btn-sm`;
бейджи `.badge .b-*` (статусы доменов/страниц); таблицы `.wrap`+`table`+`td.num`;
станции `.station`(`.plate`+`details.what`+`.go`); воронка `.funnel .fgroup .fcell.gate`;
легенда `details.legend`; чипы `.chip`; шаги `ol.steps`; карточки сайтов `.grid .scard`;
чеклист `ul.check`; строка действий `.row`+`.sep` (разделитель-черта между группами кнопок
в одной карточке, см. блок действий `/domains`).

**`.progress` УДАЛЁН** (ветка `feat/panel-m1-observability`, 2026-07-13). Старую безымянную
полосу (`#prog` в `domains.html`/`autopilot.html`, поллер снесённого `GET /run/{job}/progress`)
целиком заменила карточка задачи ниже. Класс мёртв — не заводить его заново, ссылок на него не
осталось нигде в `templates/`.

**Карточка задачи** (`.job`/`.job-head`/`.job-chips`/`.job-bar`/`.job-track`) — единственный
способ увидеть, что делает длинный прогон (discovery/score/recheck/sweep, включая автопилотный
свип из процесса worker). Рисуется общим JS-поллером `GET /api/jobs/live` (`base.html`, функции
`jobCard`/`renderMachine`) на трёх экранах с `#machine` — Пульт, `/domains`, `/autopilot`; на
остальных экранах — только тонкая версия `.mbar`/`#mbar` в шапке (полоса+процент, без чипов),
чтобы было видно, что машина работает, откуда угодно, но не ломать layout чужого экрана.
Левая полоса-акцент красится по статусу: синяя = идёт, красная `.job.err` = упала или оборвалась
(нет обновлений дольше `STALE_MIN`), серая `.job.cancelled` = остановлена человеком кнопкой
«✕ Отменить». Чипы стадий `.chip-st` — пять состояний: `done` (✓, серая заливка, стадия
пройдена), `active` (●, залит акцентом, стадия идёт сейчас), `pending` (○, контур, ещё не
дошли), `skip` (притушен — стадия выключена тумблером в `/settings`, а не сломана, поэтому не
исчезает совсем), `fail` (✕, красный — та самая стадия, на которой прогон встал; не «первая
после последней done», а именно ЗАПИСАННАЯ стадия останова).

**`.blind`** — предупреждение «оценён вслепую» строкой внутри инбокса `/domains`: домен
получил score, но история НЕ проверена (`score_breakdown.errors` содержит `wayback:`/`rkn:`/
`blacklist:`/`searxng:`). Жёлтый текст на жёлтой подложке + кнопка «▶ перепроверить» — это
плашка ВНУТРИ строки, не левый бордер: левый бордер уже занят `tr.urgent`.

**`tr.urgent`** — срочность дропа в инбоксе `/domains`: левая полоса акцентного цвета на
строке домена, чей `acquire_deadline` ближе 3 дней. Красится ТОЛЬКО при близком дедлайне, не
при любом непустом — иначе полоса легла бы на все backorder-домены разом (у них дедлайн есть
всегда) и колонка срочности перестала бы что-либо выделять.

**`.modal`/`.why-grid`/`.why-foot`** — разбор причин отказа: нативный `<dialog>`, открывается
кликом по плитке «отклонено» на `/domains`. `.why-grid` — таблица причин с долями по каждому
`reject_reason`; `.why-foot` — легенда, делящая причины на две природы: «режет порог» (можно
ослабить в `/settings`: `low_rd`/`too_young`/`low_score`) и «грязь/РКН» (не трогать:
`history_dirty`/`rkn`/`blacklist`/`feed_flag`), плюс `not_acquirable` («занят» — ни то ни
другое, чужая покупка).

## Правило шильдика (прогрессивное раскрытие)

Каждый контрол подписан ровно тем, что он делает — в три уровня:
1. **Всегда** — короткий точный лейбл на контроле.
2. **Наведение** — одна фраза в `title=`.
3. **Клик** — полный абзац/причины в `<details>` (`details.what`, `details.legend`).

Не удалять пояснения ради лаконичности — прятать за уровень 2/3.

**Поправка (2026-07-12, спека `2026-07-12-panel-m1-observability-design.md` §9).** Для ГРУППЫ
контролов, объединённых одной карточкой (блок действий M1: «Найти дропы» / «Запустить» /
«Перепроверить занятость»), три уровня на практике схлопываются в два: короткое пояснение видно
**всегда** серой строкой рядом с группой (не только в `title` у каждой кнопки по отдельности), а
полный абзац — в ОДНОМ общем `<details class="what">` на всю карточку, не в `<details>` у
каждого контрола. `title=` у отдельных кнопок при этом остаётся — уровень 2 не исчезает, просто
уровень 3 объединяется на группу. Три станции старого `/domains` схлопнулись в одну карточку
действий именно по этому правилу (см. коммит `c0b6d23` — T6, `git show c0b6d23`).

## Проверка правок

Правки дизайна проверяются глазами: локальный serve (`.venv/bin/uvicorn app.main:app`
или `docker compose up`) + Playwright-скриншоты всех экранов. Регресс разметки ловит
`pytest backend/tests/` (TestClient рендерит шаблоны). Запрещено: тёмная/индустриальная
тема, тёплая палитра, расширение CSS-контракта inline-стилями в контент-шаблонах.
