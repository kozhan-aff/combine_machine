# РКН / ЕАИС реестр запрещённых сайтов — референс интеграции

> Бесплатный **hard-reject** сигнал для скоринга доменов (**M1 Domain Intelligence**,
> `integrations/rkn.py` → `services/domain.py`, стадия history/risk — см. `docs/DONORS.md`).
> Пара к `docs/api/blacklist.md` (правило проекта: «не в РКН, не в блэклистах» — `CLAUDE.md`).
> Всё, что помечено **verified 2026-07-05**, проверено живым запросом; остальное — **UNVERIFIED**.

## Назначение — hard-reject

Если кандидат-домен есть в **ЕАИС** (Единая автоматизированная информационная система
«реестр запрещённых сайтов», ведёт Роскомнадзор) — он заблокирован на территории РФ у всех
операторов связи. Для VPN-affiliate-сайта под RU-гео это мёртвый актив: страницы недоступны
из России. Поэтому сигнал **бинарный и жёсткий**:

```
if domain in RKN_EAIS_registry:  → rkn_listed = True → auto-reject (hard fail history-гейта)
```

Это не «сок» и не качество — это чистота истории. Проверка бесплатна, работает офлайн,
без ключей и без рейт-лимита (если гоняем локальный дамп, см. «Рекомендуемый подход»).

## Рекомендуемый подход

**Локальная membership-проверка против скачанного дампа. НЕ per-domain API.**

Почему не API: единственный «живой» официальный (ЕАИС) — с CAPTCHA; rublacklist нестабилен и
сам блокируется; любой per-domain HTTP — это рейт-лимит, латентность и внешняя зависимость на
горячем пути скоринга. Локальный `set` — O(1) на проверку, офлайн, без квот, идемпотентно.

**Стратегия рефреша:**
1. Cron/воркер раз в сутки качает `RKN_DUMP_URL` (по умолчанию `dump.csv.gz` из z-i).
2. Парсит первую строку `Updated: ...` → сохраняет `dump_updated_at`.
3. **Guard свежести:** если `now - dump_updated_at > RKN_DUMP_MAX_AGE_DAYS` (напр. 3) — **алерт**
   и `rkn_check_degraded=true`. Учитывая заморозку z-i, guard сработает сразу → это фича, не баг:
   он заставит переключить `RKN_DUMP_URL` на живой источник.
4. Строит `set` заблокированных registrable-доменов (нормализация ниже), кэширует в память/на
   диск (pickle/SQLite). Атомарная замена (скачал во временный файл → распарсил → swap).
5. `services/domain.py` дёргает `is_rkn_listed(domain)` из памяти.

**Приоритет источников (конфиг):** z-i `dump.csv.gz` (полнота) → при заморозке/недоступности
fallback на `antizapret domains-export.txt` (живой, но, возможно, неполный) → rublacklist API
только как ручной спот-чек. Идеально — качать оба и OR'ить (полнота + свежесть).

**Бюджет:** ~26 MB скачивания в сутки, ~250 MB RAM на set, единицы секунд на парсинг. Бесплатно.

## Правила матчинга

Кандидат — это **registrable-домен** (eTLD+1), который мы собираемся выкупить, напр. `example.ru`.
Реестр содержит записи по домену, **wildcard-домену** (`*.example.ru`), URL и IP. Матчим так:

**Нормализация записей дампа (при построении set):**
1. Декодировать файл как **cp1251** (иначе кириллица/IDN битые).
2. Взять колонку 2 (домен). Срезать ведущий `*.` (`*.example.ru` → `example.ru`).
3. `.strip().lower()`, убрать ведущую точку.
4. **IDN:** записи `.рф`/кириллические лежат в Unicode-кириллице (cp1251). Привести обе стороны
   к **одному виду** — рекомендуется ASCII/punycode через `idna`
   (`idna.encode("0-плюс.рф").decode()` → `xn--...-...` ). Кандидат-домен тоже энкодить в punycode
   перед проверкой. (Наивный `str.encode("idna")` спотыкается на пустых лейблах/подчёркиваниях —
   лучше пакет `idna`.)
5. (Опц.) Из колонки 3 (URL) вытащить host'ы и тоже добавить (`urlparse(u).hostname`), чтобы
   ловить точечные URL-блокировки, где колонка 2 пуста.
6. IP-only строки (колонка 2 и 3 пусты) для доменной проверки игнорировать — мы скорим домен,
   не IP.

**Проверка кандидата `d`** (нормализованного так же — lower, punycode, без `*.`):
- **Точное совпадение:** `d in blocked_set` → listed.
- **Родительский домен:** если любой суффикс `d` (по лейблам) есть в `blocked_set` — listed.
  Т.е. запись `*.example.ru` (в set как `example.ru`) блокирует и `example.ru`, и `sub.example.ru`.
  ```
  labels = d.split('.')
  for i in range(len(labels)-1):
      if '.'.join(labels[i:]) in blocked_set: → listed
  ```
  На практике покупаем сам eTLD+1, так что решает п.1; п.2 нужен, если оцениваем сабдомен или
  чтобы поймать `d = example.ru` при записи `www.example.ru`.
- **Wildcard уже «схлопнут»** на шаге нормализации (мы срезали `*.`), поэтому отдельной логики
  для `*` не требуется — родительский-суффикс матч её покрывает.

**Итог:** `rkn_listed = exact_match OR parent_suffix_match`.

## Пример (Python, псевдо → рабочий)

```python
import csv, gzip, io, idna, httpx
from datetime import datetime, timezone

RKN_DUMP_URL = "https://raw.githubusercontent.com/zapret-info/z-i/master/dump.csv.gz"

def _norm(host: str) -> str | None:
    host = host.strip().lstrip("*.").strip(".").lower()
    if not host:
        return None
    try:                                   # к единому виду: ASCII/punycode
        return idna.encode(host).decode("ascii")
    except idna.IDNAError:
        return host                        # уже ASCII или мусор — как есть

def load_blocked(url: str = RKN_DUMP_URL) -> tuple[set[str], datetime | None]:
    raw = httpx.get(url, timeout=120).content
    data = gzip.decompress(raw)            # dump.csv.gz → байты
    text = data.decode("cp1251")           # ⚠️ НЕ utf-8
    lines = text.splitlines()

    updated = None
    if lines and lines[0].startswith("Updated:"):
        # "Updated: 2025-10-01 10:25:04 +0000"
        updated = datetime.strptime(lines[0][8:].strip(), "%Y-%m-%d %H:%M:%S %z")
        lines = lines[1:]                  # метадату — из данных вон

    blocked: set[str] = set()
    for row in csv.reader(lines, delimiter=";"):
        if len(row) < 3:
            continue
        if (n := _norm(row[1])):           # колонка 2 — домен (уже без *.)
            blocked.add(n)
        for u in row[2].split("|"):        # колонка 3 — URL: вытащить host
            if "//" in u and (h := _norm(u.split("//", 1)[1].split("/", 1)[0])):
                blocked.add(h)
    return blocked, updated

def is_rkn_listed(domain: str, blocked: set[str]) -> bool:
    d = _norm(domain)
    if not d:
        return False
    labels = d.split(".")
    return any(".".join(labels[i:]) in blocked for i in range(len(labels) - 1))

# --- использование ---
blocked, updated = load_blocked()
if updated and (datetime.now(timezone.utc) - updated).days > 3:
    log.warning("RKN dump stale: %s — switch RKN_DUMP_URL", updated)  # z-i заморожен!
rkn_listed = is_rkn_listed("example.ru", blocked)   # → hard-reject если True
```

Fallback на antizapret (уже нормализовано, UTF-8):
```python
lines = httpx.get("https://antizapret.prostovpn.org/domains-export.txt").text.splitlines()
blocked = {_norm(x) for x in lines if x} - {None}
```

## Лимиты и подводные камни

- **z-i заморожен с 2025-10-01** (см. верх). Без freshness-guard получишь ложно-«чистые» домены,
  заблокированные после этой даты. Держи `RKN_DUMP_URL` конфигурируемым.
- **cp1251, не UTF-8.** `data.decode("utf-8")` упадёт/побьёт кириллицу и все `.рф`/IDN-записи.
- **IDN в Unicode-кириллице**, а не punycode — приводи обе стороны к одному виду (`idna`), иначе
  `xn--...`-кандидат никогда не совпадёт с `0-плюс.рф` в дампе.
- **Память:** ~1–1.8M строк-доменов → ~200–300 MB на `set`. Если тесно — SQLite/Bloom-filter.
- **`;` внутри URL** (query-string) теоретически ломает naive `split(";")` — используй
  `csv.reader(delimiter=";")` (как в примере) и бери колонку 2 по индексу.
- **IP-only записи** (колонка 2 пуста) — не про домены; для доменной проверки пропускать. Если
  когда-нибудь надо ловить блок по IP хостинга — это отдельная IP-membership логика (см.
  `rkn-exporter`: резолвит домен в IP и сверяет с `/api/v3/ips/`).
- **rublacklist нестабилен** (на 2026-07-05 недоступен; сам под блокировкой в РФ). Не строить на
  нём горячий путь.
- **antizapret** — обработанный сторонний список, возможно неполный и без метки свежести
  (**UNVERIFIED**). Хорош как кросс-чек, не как единственный источник.
- **Юридическая чувствительность.** Данные публично зеркалируются, но это реестр запрещёнки —
  использовать только внутренним скорингом, не хостить/не реэкспортить.
- **Ложные срабатывания.** Массовые авто-блокировки ФНС (гемблинг) могли зацепить домен, который
  теперь дропнулся и «чист» по контенту, но запись в реестре остаётся → для нас это всё равно
  hard-reject (в РФ домен недоступен), что корректно для RU-гео.
