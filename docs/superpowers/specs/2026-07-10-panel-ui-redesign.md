# Дизайн-спека: переосмысление панели КОМБАЙН (холодная минимальная CMS)

**Дата:** 2026-07-10
**Статус:** утверждена к реализации
**Область:** только визуальный слой — `backend/app/templates/` (весь CSS-контракт в `base.html`) + новый `docs/DESIGN.md`. Бэкенд/роуты/логика НЕ трогаются.

## Проблема

Текущая панель — светлая CMS (тёплая бумага `#f6f4ef`, оранжевый акцент `#e05e10`,
Golos Text). Функционально работает, дизайн-контракт держится, но пользователь
отвергает её как «не от панели, а наспех сделанное ИИ». Диагноз после аудита
`base.html` + шаблонов: **вес наспешности не в цвете, а в вербозности.**

- Каждая станция (`.station`) несёт абзац `.what` в 3–4 строки — всегда развёрнут.
- Сайдбар: под каждым из 8 пунктов вечная подпись `.sub` (простыня 248px).
- Воронка пульта: 9 плиток, по 3 строки текста на плитку, высота ~350px.
- Тёплая палитра + скругления + мягкие тени добавляют «уютности», которая читается
  как несерьёзность для рабочего инструмента управления конвейером.

Инвариант CLAUDE.md «каждый контрол подписан ровно тем, что он делает» («шильдик»)
выродился в стену текста, показываемую всю сразу. Чиним это, не отменяя инвариант.

## Цель

Лаконичная, холодная, плотная панель управления в духе Linear / GitHub / Vercel:
структура видна с одного взгляда, текст-пояснения доступны прогрессивно (по наведению
и клику), а не вываливаются стеной. Инварианты продукта и дизайн-контракта сохраняются.

## Принципы

1. **Холодно и нейтрально.** Нейтрально-серый фон, чистый белый на панелях, один
   холодный синий акцент вместо оранжевого. Никакой «бумаги» и тепла.
2. **Плотность.** Меньше вертикали на тот же объём данных — компактнее сайдбар,
   воронка-лента, свёрнутые пояснения.
3. **Прогрессивный шильдик.** Инвариант «каждый контрол подписан» остаётся, но в три
   уровня: всегда — короткий точный лейбл; наведение (`title`) — фраза что делает;
   клик (`<details>`) — полный абзац с причинами/кодами.
4. **CSS-контракт един.** Весь визуал в `base.html`; шаблоны — только семантика +
   классы контракта. Не расширять контракт inline-стилями на месте (текущие шаблоны
   этим грешат — по ходу подчистить очевидное, без голдплейта).

## Дизайн-токены (`:root` в base.html)

Замена текущего блока `:root`. Значения — финальные.

### Цвет — холодная нейтраль + синий акцент

```
--bg:#f7f8fa;        /* нейтральный холодный фон (был тёплый #f6f4ef)     */
--panel:#ffffff;     /* карточки/панели — чистый белый                    */
--panel2:#f2f4f7;    /* вторичный фон: шапки таблиц, hover, свёрнутое      */
--line:#e6e8ec;      /* основной разделитель (холодный, был тёплый)        */
--line2:#d5d9e0;     /* усиленный бордюр (инпуты, кнопки)                  */
--ink:#1a1d23;       /* основной текст (почти чёрный, холодный)            */
--mut:#5c626d;       /* вторичный текст                                    */
--dim:#9aa1ad;       /* третичный/подписи                                  */

--acc:#2563c9;       /* АКЦЕНТ: холодный синий (был оранжевый --amber)     */
--acc2:#1d4ed8;      /* акцент hover/darker                                */
--acc-soft:#e8f0fe;  /* мягкая заливка акцента (бейджи/активное)           */

--ok:#137a43;   --ok-soft:#e3f4ea;    /* одобрено/успех — холодный зелёный */
--bad:#c62b2b;  --bad-soft:#fbe9e9;   /* reject/ошибка                     */
--warn:#9a6b0f; --warn-soft:#fbf1da;  /* внимание/жёлтый                   */
--info:#0e7490; --info-soft:#e0f2f7;  /* инфо/бирюза (для discovered/echo) */
--violet:#6d3fc4; --violet-soft:#efe9fb; /* purchased/выкуп                */
```

**Роль акцента меняется:** синий `--acc` — это «система/действие» (primary-кнопки,
активный пункт меню, ссылки, фокус). Статусные цвета (ok/bad/warn/violet) остаются
семантическими. Оранжевого в палитре больше нет.

### Типографика — нейтральная UI + моно для данных

```
--ui:"Inter","Segoe UI Variable","SF Pro Text",system-ui,-apple-system,sans-serif;
--mono:"JetBrains Mono","Cascadia Mono",Consolas,"SF Mono",ui-monospace,monospace;
--brand:"Unbounded","Inter",system-ui,sans-serif;   /* только логотип */
```

- **UI/текст:** Inter (был Golos Text) — канонический шрифт панелей управления,
  холодный, плотный, читаемый в мелком кегле. Для дата-инструмента restraint = дизайн.
- **Данные/числа/коды:** JetBrains Mono остаётся (табличные цифры, коды, RD/score).
- **Логотип:** Unbounded остаётся (единственная «характерная» деталь — акцент бренда,
  не UI). Google Fonts с фолбэком на системные, как сейчас.
- Обновить `<link>` Google Fonts: `Inter:wght@400;500;600;700` вместо `Golos Text`.

### Форма — суше

```
--r:8px;            /* радиус (был 10px — чуть острее, «панельнее»)        */
--r-sm:6px;
--shadow:0 1px 2px rgba(20,25,35,.04);   /* ЕДИНАЯ лёгкая тень (была двойная мягкая) */
--shadow-none:none;
```

Тени почти убрать: панель держится на бордюре `--line`, не на «парении». Одна лёгкая
тень для приподнятых элементов (карточки), не двухслойная тёплая.

## Архитектура макета

### Сайдбар — компактнее (248px → 200px), без вечных подписей

- Убрать `.rl .sub` из потока (описание пункта уходит в `title` атрибут ссылки).
- Пункт = моно-индекс `.n` + название. Высота пункта падает вдвое.
- `.brand` + `.brand-sub` остаются (логотип Unbounded + одна строка).
- Блок `.gates` внизу («выкуп подтверждает человек / публикация только edited») —
  остаётся: это напоминание про хард-гейты, ценное. Сузить кегль.
- Активный пункт: `--acc-soft` фон + `--acc` текст (был amber-soft).
- Мобильный брейкпоинт (<900px, сайдбар в строку) — логика та же, цвета обновить.

### Контент

- `main` max-width 1440 → **1360px** (чуть плотнее), паддинги те же.
- Мягкое появление секций (`@keyframes rise`) — оставить, безобидно и снижает
  ощущение «статичной простыни».

## Компоненты (обновление CSS-контракта)

Правки внутри существующих классов — имена классов НЕ меняем (шаблоны от них зависят),
меняем только их оформление и, для станций/сайдбара, разметку-обёртку.

### Кнопки
- `.btn-amber` → перекрасить в синий primary (класс-имя оставить ради обратной
  совместимости шаблонов; ИЛИ ввести алиас `.btn-acc` и заменить в шаблонах — см.
  «Открытый вопрос» ниже). Primary = `--acc` фон, белый текст.
- `.btn-ok`/`.btn-bad`/`.btn-buy` — перекрасить под новые холодные ok/bad/violet.
- Убрать тёплый hover, острее радиус `--r-sm`.

### Бейджи статусов (`.b-*`)
- Перевести на новые токены: discovered→info(бирюза), scored→acc(синий, был amber),
  approved→ok, rejected→bad, purchased/purchasing→violet, live/published→ok-тёмный,
  provisioning→warn, content→info, draft/default→нейтраль, edited→acc.
- Форма компактнее: `padding:2px 9px`, тот же pill.

### Станции (`.station`) — главное изменение вербозности
Разметка станции меняется: абзац `.what` оборачивается в `<details class="what">`
со свёрнутым по умолчанию `<summary>зачем это</summary>`.

```html
<div class="station">
  <div class="plate">▶ Проверка — оценить очередь <span class="mod">M1b</span></div>
  <details class="what">
    <summary>зачем это</summary>
    <div class="what-body">Прогоняет discovered-домены через воронку…</div>
  </details>
  <div class="go"><!-- кнопки как есть --></div>
</div>
```

CSS: `details.what > summary` — мелкая серая строка-«зачем это» (как `details.legend`),
`.what-body` — раскрытый абзац. Свёрнутая станция = заголовок + кнопка, ~3 строки.

### Воронка пульта (`.funnel`) — лента, сгруппированная по модулям
Переверстать `dashboard.html`: вместо 9 равных `.stat` со стрелками `▸` —
три группы-сегмента (M1 ДОБЫЧА / M2 ВЫКУП / M3–M5 САЙТЫ), внутри группы компактные
ячейки «число + одно слово». Служебные пояснения (нынешние 3-и строки `.k`) уходят
в `title` ячейки.

- Число крупным моно, одна подпись-слово под ним.
- **Гейт-индикатор:** там где что-то ждёт человека (scored / подтверждение выкупа /
  draft-редактура) — точка `●` цветом `--acc`. Заменяет текущий класс `.hot`
  (перекрасить `.stat .v.hot` с amber на acc, добавить точку-маркер).
- Высота ленты ~110px против ~350px.
- Кликабельность сохраняется (каждая ячейка — ссылка в список стадии).
- Ниже — «Что дальше» (`ol.steps`, оставить) и карточки сайтов (`.scard`, оставить,
  перекрасить).

### Таблицы (`.wrap`/`table`)
- Оставить структуру. Обновить цвета: шапка `--panel2`, hover `--panel2`,
  бордюры `--line`. Моно-шапка остаётся (хороший «панельный» приём).
- `.meter` (score-бар) — перекрасить m-ok/m-mid/m-low под холодные ok/warn/bad.

### Легенда (`details.legend`), чипы (`.chips`), прогресс (`.progress`)
- Перекрасить под токены. Структуру не менять — легенда `<details>` уже эталон
  прогрессивного шильдика, на неё равняем станции.
- `.progress .fill` — из amber в `--acc`.

## Что НЕ меняется (инварианты)

- **Дизайн-контракт CLAUDE.md:** светлая CMS (остаётся светлой), принцип шильдика
  (остаётся, становится прогрессивным), CSS-классы только из `base.html`, UI на русском.
  Тёмное/индустриальное по-прежнему запрещено — новая палитра светлая.
- **Функционал:** ни один роут, обработчик формы, `name=`/`action=` не трогается.
  Только представление.
- **Хард-гейты** (деньги/редактура) — вне области, это UI-слой.
- **Доступность:** `:focus-visible` остаётся (перекрасить в `--acc`), контраст
  текста на фоне держать ≥ AA (проверить `--mut`/`--dim` на `--bg`),
  `prefers-reduced-motion` уважать (уже есть).

## Открытый вопрос (решить в плане реализации)

**Класс primary-кнопки:** `.btn-amber` встречается в шаблонах десятки раз. Два пути:
(а) оставить имя `.btn-amber`, просто перекрасить в синий — ноль правок в шаблонах,
но имя врёт; (б) ввести `.btn-acc`, заменить в шаблонах — честное имя, но diff шире.
Рекомендация: **(б)** — семантически честно, замена механическая (sed по templates),
имя `amber` в холодной палитре будет сбивать будущего разработчика. То же для
`.stat .v.hot` → оставить имя (нейтральное, «hot» = «требует внимания», не цвет).

## Вне области (YAGNI)

- Полноценный редизайн-фреймворк/дизайн-токены как отдельный CSS-файл или сборка —
  контракт остаётся инлайн в `base.html` (проект — один `base.html`, так проще).
- Тёмная тема — не просили, светлая CMS по контракту.
- Компонентная библиотека/Storybook — избыточно для Jinja-панели на одного оператора.
- Изменение информационной архитектуры (набор экранов, навигация) — экраны те же.
