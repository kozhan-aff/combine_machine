# Спек 1 — Мозг M1: приобретаемость как гейт воронки

> Дизайн-документ. Терминальное состояние брейнсторма — передача в writing-plans.
> Часть 1 из 3 (Мозг M1 · Лицо · Комбайн). Локализация панели, визуальный прогресс-бар
> и автономия — отдельные спеки, здесь НЕ проектируются (см. §Границы).

## Цель

Машина сейчас скорит «хороший ли донор?», но нигде не спрашивает «а могу ли я его
вообще получить — каким лейном, до какого дедлайна?». Дорогие проверки (whois-возраст,
Wayback-история) тратятся в том числе на сырые имена из списков удаления, которые купить
нельзя. Этот спек вводит **приобретаемость как ранний гейт воронки**: в скоринг и на
дорогие проверки идёт только то, что реально приобретаемо, с проставленным лейном выкупа
и дедлайном. Это качество входа для будущей автономии — без него комбайн будет автономно
тащить мусор.

## Принятые решения (из брейнсторма)

1. **Два лейна выкупа:** `bid` (ценные дропы на backorder — ставка, дедлайн) и `free`
   (свободные к немедленной регистрации, подтверждённые whois). Собираем и скорим только
   то, что попадает в один из лейнов.
2. **backorder — единственный bid-источник.** Аукцион / nic.ru НЕ заводим. nic.ru
   рассмотрим позже только если окажется, что там есть реально свободные (не аукционные)
   домены → тогда подключим как ещё один free-источник.
3. **whois-пробой решает лейн:** один вызов даёт и доступность (свободен/занят), и
   возраст. Занятый сырой домен, которого нет на backorder, → отклоняется как
   `not_acquirable` до дорогих проверок.
4. **cctld демотируется** сортировкой `RD desc nulls-last` (у сырых имён RD нет →
   пробиваются последними) + жёсткий кап whois-вызовов на прогон.

## §A. Модель лейнов

| Лейн | Источник | Как берём | Дедлайн | Сигнал приобретаемости |
|------|----------|-----------|---------|------------------------|
| `bid` | backorder | ставка (ловля на удалении) | `delete_date` из фида | лейн из источника (домен в фиде backorder = биддируемый) |
| `free` | cctld / reg.ru / sweb | регистрация первым (optimizator) | нет | whois: запись отсутствует → домен свободен |
| — (reject `not_acquirable`) | cctld / reg.ru / sweb | никак | — | whois: домен занят, и его нет на backorder |

Принцип: **лейн определяется до дорогих проверок.** Нет лейна → в T2/T3 и в скор не идём.

## §B. Данные

### Domain — новые поля
- `lane: str | None` — `bid` | `free` | `null` (пока не определён).
- `acquire_deadline: datetime | None` — дедлайн ловли/аукциона (из backorder `delete_date`).
- `acquire_price: numeric | None` + `price_checked_at: datetime | None` — цена выкупа и когда
  сверялась. Для backorder заполняется из тарифного эндпоинта (базовая цена бэкордера .ru,
  см. «Цена бэкордера»), обновляется кнопкой. Живую аукционную цену конкретного домена бесплатный фид не
  публикует — показываем базовую; per-domain цену используем только если найдётся на живом
  API (кандидат — поле `x_value` фида). Для free-лейна — стоимость регистрации (позже).
- Сохранять выбрасываемый сейчас фид-сигнал: `visitors: int | None`, `tic: int | None`
  (Яндекс ТИЦ). Информационно, вес в скоре 0 — на будущее, как DR.
- `reject_reason` расширяется значением **`not_acquirable`**.

Полный enum `reject_reason`:
`low_rd | feed_flag | too_young | rkn | blacklist | history_dirty | low_score | not_acquirable`

### ScoringSettings — новый порог
- `max_whois_per_run: int` (дефолт `200`) — сколько whois-вызовов максимум за один прогон
  проверки. Клампится в разумных границах (напр. 0…5000). Дефолт в `scoring_config.py`.

### Миграция
`0003_acquirability` — добавляет `lane`, `acquire_deadline`, `acquire_price`, `visitors`,
`tic` в `domains` (все nullable, старые строки корректны) и `max_whois_per_run` в
`scoring_settings` (server_default 200). Накатывается кнопкой «Обновить из git»
(`alembic upgrade head`), как 0002.

## §C. Интеграция whois — доступность + возраст одним вызовом

Расширяем A-Parser whois-клиент. Сейчас есть `whois_created(domain) -> datetime|None`
(возвращает дату регистрации). Добавляем единый пробой:

```
whois_probe(domain) -> {"available": bool | None, "created": datetime | None}
```
- `available=True` — whois вернул «нет записи» (домен свободен к регистрации). `created=None`.
- `available=False` — домен зарегистрирован. `created` = дата регистрации (или `None`, если
  запись есть, но дату не распарсили).
- `available=None` — whois недоступен/сбой транспорта (не путать со «свободен»).

`whois_created` остаётся как есть (обратная совместимость), либо становится тонкой обёрткой
над `whois_probe(...)["created"]`.

**Открыто (снять на живом прогоне):** точные маркеры «свободен» в ответе A-Parser Net::Whois
для .ru/.рф (напр. `No entries found`, `Not found`, пустой ответ). Парсер держим устойчивым
к формулировкам; список маркеров — в одном месте, легко дополнить с живого ответа.

## §D. Воронка — приобретаемость встроена в T1

Ступени (изменения только в T1; T0/T2/T3 по сути прежние, но T2/T3 идут только на
приобретаемых):

- **T0 — фид (дёшево):** RD/флаги как сейчас (`low_rd` / `feed_flag`).
- **T1 — приобретаемость + возраст (whois):**
  - Источник `backorder` → `lane = bid`, `acquire_deadline` уже проставлен на discovery.
    whois вызывается только ради `created` (возраст). `available` для лейна не нужен.
  - Источник сырой (`cctld` / `reg_ru` / `sweb`) → `whois_probe`:
    - `available=True` → `lane = free`, возраст добираем из Wayback (у свободного домена
      whois-даты нет — используем существующий Wayback-fallback возраста).
    - `available=False` → **reject `not_acquirable`** (занят, не на backorder — купить
      нельзя). Ранний выход, T2/T3 не трогаем.
    - `available=None` (whois-сбой) → **не отклоняем** по not_acquirable (это был бы
      ложный отказ на транзиентном сбое): домен уходит в ручной разбор (`scored`) с
      пометкой «приобретаемость не проверена (whois недоступен)», дорогие проверки не
      тратим; перепроверится следующим прогоном. (Авто-ретрай — забота Спека 3.)
  - Гейт возраста `too_young` работает как раньше поверх полученного возраста.
- **T2 — риск:** РКН / Spamhaus / indexed_echo — как сейчас, **только на домене с лейном**.
- **T3 — Wayback-история:** как сейчас, **только на выживших T0–T2 с лейном**.
- **Скор** (`compute_score` + рантайм-пороги) — без изменений логики, `lane`/`acquire_*`
  на скор не влияют (это гейт приобретаемости, а не сигнал качества).

Инвариант сохраняется и усиливается: **дорогой Wayback — только для домена, который
проверен как приобретаемый И прошёл T0–T2.** `compute_score` остаётся чистой.

## §E. Стоимость whois и демотация cctld

whois — сетевой вызов на домен. Сырой cctld-список может быть большим, поэтому:

- **Кап `max_whois_per_run`** (§B): за прогон проверки whois вызывается не более N раз.
  Домены сверх бюджета остаются `discovered` до следующего прогона (не отклоняются).
- **Порядок уже правильный:** `score_pending` сортирует `referring_domains desc nulls-last`
  → backorder-домены (с RD) пробиваются первыми, сырой cctld (RD = null) — только если
  остался бюджет. Отдельной логики демотации cctld не нужно, сортировка её обеспечивает.
- Прогресс-полоса сообщает исчерпание бюджета: «проверено X, whois-бюджет исчерпан, осталось Y».

**Известное ограничение (документируем, не чиним здесь).** cctld dellist — это домены,
которые *будут* удалены; пробивая их сегодня, whois чаще вернёт «занят» → `not_acquirable`.
Свободными они станут только после своей даты удаления. В однопроходной модели
whois-пробоя (без watch-планировщика — его мы сознательно не берём) ценность cctld
раскрывается в основном на повторных прогонах после дат дропа. Основной выход дают
backorder (bid, с RD и дедлайном) и реально-свободные листинги. Возможное уточнение на
будущее: парсить дату удаления из cctld и пробивать/впускать домен только после неё.

## §F. Discovery — сохранить лейн-сигнал источника

- `normalize_row` (backorder) начинает сохранять то, что сейчас выбрасывает:
  `delete_date → acquire_deadline`, `visitors`, `yandex_tic → tic`. Флаги (rkn/judicial/
  block) — как сейчас в `feed_flags`.
- backorder-строки помечаются `lane = bid` уже на вставке (источник = биддируемый), и им
  проставляется `acquire_price` из кэша тарифа (см. «Цена бэкордера»), если он есть; иначе `null` до обновления.
- Сырые источники (cctld/reg.ru/sweb) вставляются как сейчас, `lane = null` — лейн им
  проставит whois-пробой на T1.
- Дедуп по домену без изменений (больший RD выигрывает; backorder первым — его
  `feed_flags`/`acquire_deadline` переживают ничью).

## §G. Панель (только новые контролы; массовая русификация — Спек 2)

Все новые строки — сразу живым русским (не плодим англицизмы).

- **Бейдж источника у имени домена** (моно, в стиле шильдика, отдельно от значка ⌛
  Wayback-ссылки, который уже стоит):
  - `[bo]` **оранжевый** — backorder, лейн ставка. Единственный оранжевый акцент = ценное.
  - `[cc]` / `[rg]` / `[sw]` **серые** — cctld / reg.ru / sweb, кандидаты в free.
  - `title`: «источник: backorder · лейн: ставка · дедлайн 08.07».
- **Колонки** `лейн`, `дедлайн`, `цена` — как разворот деталей (бейдж+тултип может хватать;
  колонки оставляем для сортировки/сканирования). Цена — для backorder-строк, с подсказкой
  «базовая цена бэкордера, обновлено DD.MM» (`price_checked_at`).
- **Кнопка «Обновить цены бэкордера»** (на `/domains`, рядом с прогоном) — перечитывает тариф
  и проставляет `acquire_price`/`price_checked_at` всем backorder-доменам. Дёшево (один
  публичный JSON), денег не тратит.
- **`/domains` по умолчанию — только приобретаемые:** домены со статусом-причиной
  `not_acquirable` скрыты; фильтр «показать все» их открывает (не удаляем — видны с причиной).
- **Кнопки полного прогона** (рядом с батч-запуском «за N»):
  - «Проверить весь пул» — score без лимита 5 (бюджет ограничен `max_whois_per_run`;
    домены сверх бюджета остаются на следующий прогон). Прогресс-полоса показывает домен
    в работе и исчерпание бюджета.
  - Поиск дропов уже одноразовый («собрать всё») — отдельная кнопка не нужна, но подпись
    сверяем с реальностью.
- **`/settings`** — ползунок `max_whois_per_run` с живым счётчиком-подписью, как у прочих
  порогов; зеркалит воронку (сколько доменов уложится в бюджет).

Дизайн новых элементов проверяем глазами (локальный serve + Playwright-скриншоты).

## Цена бэкордера — получить и обновить

- `BackorderClient.get_tariffs()` (сейчас заглушка) реализуется: GET публичного
  `price_ru_backorder.ru.json`, парс → базовая цена (`price_id`, `period_id`, сумма). Без auth.
- Сервис `refresh_backorder_prices()`: читает тариф, проставляет `acquire_price` +
  `price_checked_at` всем backorder-доменам. Тариф кэшируется (короткий TTL / по кнопке),
  чтобы не дёргать на каждый прогон.
- discovery ставит `acquire_price` из кэша тарифа при вставке (§F); кнопка «Обновить цены»
  и последующий refresh обновляют.
- **Честная граница:** это *базовая цена бэкордера* (сколько стоит поставить ловлю), не живая
  аукционная цена контестного домена — её бесплатный фид не отдаёт. Если на живом API
  найдётся per-domain цена (кандидат — `x_value` фида) — используем её; иначе базовая.
  Проверить на первом прогоне (см. §J).

## §H. Тесты (оффлайн, SQLite-харнесс, сеть мокается)

- **whois_probe:** `available=True/False/None`, `created` парсится/None; обратная
  совместимость `whois_created`.
- **Воронка — лейны:**
  - backorder-домен → `lane=bid`, доходит до Wayback (при выживании T0–T2).
  - сырой + whois «свободен» → `lane=free`, возраст из Wayback-fallback.
  - сырой + whois «занят» → `reject_reason=not_acquirable`, **Wayback НЕ вызывался**
    (`wb.calls==0`) — экономия доказана.
  - сырой + whois-сбой → `scored` (ручной), не `not_acquirable`, Wayback не вызывался.
- **whois-кап:** при `max_whois_per_run=1` и трёх discovered пробивается ровно один
  (по RD desc), остальные остаются `discovered`.
- **Discovery:** `acquire_deadline`/`visitors`/`tic` сохраняются из backorder-фида;
  backorder-строка получает `lane=bid`.
- **Панель:** `/domains` по умолчанию скрывает `not_acquirable`, фильтр «все» показывает;
  бейдж источника рендерится с нужным title; «Проверить весь пул» запускает score без
  батч-лимита; ползунок `max_whois_per_run` сохраняется и клампится.
- **Цена бэкордера:** `get_tariffs` парсит замоканный JSON тарифа → базовая цена;
  `refresh_backorder_prices` проставляет `acquire_price`/`price_checked_at` только
  backorder-доменам (free/сырые не трогает); кнопка на панели дёргает refresh.
- **Индикаторы `/diag`:** A-Parser и БД появляются в списке пинга; при мокнутом сбое
  A-Parser строка = FAIL с текстом (сеть замокана, не ходим на живой бокс).
- pyflakes чистый; полный прогон детерминирован.

## §K. Фазы реализации (для writing-plans)

1. Данные: колонки Domain (`lane`, `acquire_deadline`, `acquire_price`, `price_checked_at`,
   `visitors`, `tic`), `reject_reason += not_acquirable`, `max_whois_per_run` в ScoringSettings,
   миграция `0003_acquirability`, регистрация в conftest.
2. whois: `whois_probe` (available + created) в A-Parser клиенте + парсер маркеров «свободен».
3. Discovery: сохранять `acquire_deadline`/`visitors`/`tic`, метить backorder `lane=bid`.
4. Воронка: T1 приобретаемость (лейн/`not_acquirable`/whois-fail→scored), whois-кап,
   Wayback только на приобретаемых с лейном; регрессия `wb.calls==0` на not_acquirable.
5. Цена бэкордера: `get_tariffs` + `refresh_backorder_prices`, кэш тарифа, проставление
   `acquire_price`/`price_checked_at` backorder-доменам.
6. Индикаторы `/diag`: A-Parser + БД в пинг-список.
