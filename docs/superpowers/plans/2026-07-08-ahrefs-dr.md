# Ahrefs DR/backlinks/referring-domains Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a real, live-verified Ahrefs DR/backlinks/referring-domains signal to M1
scoring (via A-Parser's `Rank::Ahrefs` preset + `RuCapcha` Turnstile-solver), replacing
the dead OpenPageRank integration, gated behind a runtime-adjustable money cap.

**Architecture:** New `AParserClient.ahrefs_probe()` method (transport + parsing, same
file as the existing `whois_probe`). `scoring.py`'s `_funnel()` calls it only for T3
survivors that the discovery feed didn't already give a referring-domains count for
(cctld/reg_ru/sweb — backorder domains skip it, they already have real RD). DR revives
the currently-dead `authority` component in the composite score (real weight now);
Ahrefs' referring-domains count feeds `rd_proxy`. A new `max_ahrefs_per_run` runtime
setting (DB-backed, `/settings` slider) caps how many paid captcha-solves happen per
scoring run — unlike `max_whois_per_run`, `0` is a legal value here (fully disables
Ahrefs enrichment; whois has no such off-switch since it's free).

**Tech Stack:** Python 3.12, FastAPI, SQLAlchemy 2.x, Alembic, pytest (SQLite offline
harness, `backend/tests/conftest.py` autouse-fixture blocks live network).

## Global Constraints

- `integrations/` = transport only, but response-format parsing lives in the same file
  as the transport call (existing pattern: `aparser.py`'s `_parse_whois_created` sits
  next to `whois_probe`; `cctld.py`'s zip-parsing sits next to its `list_dropping`).
- Secrets only in `.env`. `APARSER_PROXY_CHECKER` is not a secret but is
  environment/box-specific config — goes in `config.py`/`.env` like `APARSER_URL`.
- No new dependencies — stdlib `re` only; transport is the existing `httpx`-based
  `BaseClient`/`AParserClient`.
- Tests are offline and hermetic (SQLite, `backend/tests/conftest.py` autouse fixture
  blocks live network) — every new/changed test must pass with zero network access.
  `pyflakes` must stay clean. Test output must stay clean (no warnings).
- `Base.metadata.create_all()` builds the test DB schema directly from the SQLAlchemy
  models (confirmed in `conftest.py`) — Alembic migrations are NOT run for tests, only
  for the real Postgres box. A model column change alone unblocks tests; the migration
  file is still required for the box, but its absence never breaks the test suite.
- `/settings` UI: reuse existing CSS classes from `base.html`'s established pattern
  (`.station`, `.plate`, `.what`, `.go`, `.hint` — see the `max_whois_per_run` block in
  `templates/settings.html` for the exact shape). Do not invent new CSS classes. UI text
  in Russian ("шильдик" principle — every control is labeled with exactly what it does).
- Money/editorial hard gates (`confirmed_by_human`, `mark_edited`) are untouched — this
  is M1 scoring, not M2 acquisition or M4 publishing.
- Test count baseline before this plan: **191 passed**, `pyflakes` clean
  (`.venv/bin/python -m pytest backend/tests/ -q` / `.venv/bin/python -m pyflakes

- [ ] **Step 4: Implement `_parse_ahrefs()` and `ahrefs_probe()` in `aparser.py`**

In `backend/app/integrations/aparser.py`, add near the top (after the existing

```python
# Rank::Ahrefs resultString (Result format "$query: $rating, $bl, $domains\n"), живьём
# проверено 2026-07-08: 'wikipedia.org: 97, 3421649284, 4000871' / 'lev777.casino: none, 476, 268'
# rating может быть буквально строкой 'none' — Ahrefs не всем доменам присваивает DR,
# даже когда backlinks/referring-domains у них реальные (не 0) — отдельный null-check.
_RE_AHREFS = re.compile(
    r"^\s*\S+:\s*(?P<rating>none|\d+)\s*,\s*(?P<bl>\d+)\s*,\s*(?P<domains>\d+)\s*$",
    re.I | re.M,
)


def _parse_ahrefs(text: str) -> dict:
    """resultString '<domain>: <rating|none>, <bl>, <domains>' -> dr/backlinks/
    referring_domains. Немэтч (разметка A-Parser/Ahrefs изменилась) -> все None, не
    исключение — вызывающий код (scoring._funnel) уже привык к None-сигналам."""
    m = _RE_AHREFS.search(text or "")
    if not m:
        return {"dr": None, "backlinks": None, "referring_domains": None}
    rating = m.group("rating")
    return {
        "dr": None if rating.lower() == "none" else int(rating),
        "backlinks": int(m.group("bl")),
        "referring_domains": int(m.group("domains")),
    }
```

Add the method to the `AParserClient` class, right after `whois_probe`/`whois_created`

        2026-07-08 — см. docs/superpowers/specs/2026-07-08-ahrefs-dr-design.md).
        Дорогой вызов (платная капча) — вызывающий код решает, КОГДА его делать
