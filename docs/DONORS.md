# DONORS — методика оценки доменов и доноров

Ценность дропа = его ссылочный профиль. Но профиль профилю рознь: масса дешёвых
или спамных доноров хуже, чем горстка чистых тематических. Проверка идёт воронкой
от дешёвых сигналов к дорогим — дорогие считаем только по выжившим.

> Помни принцип проекта: домен берём за **чистую историю и релевантность**, а не за
> «сок». Google с 2026 целенаправленно гасит expired-domain abuse — грязный дроп это
> не буст, а мина. Эта воронка нужна, чтобы отсеять мины, а не чтобы найти самый жирный «сок».

## Стадия A — Intake (дёшево, все кандидаты)
- Источник: backorder (освобождаются завтра, ≥1 донор) + доступность у optimizator.
- Санити: длина/паттерн домена, TLD, явный мусор — отсечь.

## Стадия B — Метрики pre-filter (дёшево, батч)
- Тянем: **DR, referring domains (RD), backlinks, оценка орг-трафика**.
- Пороги (пример, тюнится в `scoring_config`): `RD ≥ min_rd`, `DR ≥ min_dr`.
- **RD важнее backlinks:** 10000 ссылок с 2 доменов ≪ 50 ссылок с 50 доменов.
- Провайдер: Ahrefs API / DataForSEO / checktrust (см. SERVICES.md; выбор по бюджету).

## Стадия C — Качество доноров (дорого, только выжившие)
- **Live vs lost:** живы ли ссылки сейчас (Ahrefs фильтр live / пересбор выборки).
  Профиль, где большинство «lost» — обесценить.
- **Анкоры:** посчитать доли {брендовые, URL, generic, money-keyword, иноязычные,
  adult/pharma/gambling}. Красный флаг: доля money/иноязычных/adult выше порога →
  вероятный экс-PBN/спам. Естественный профиль — преобладание брендовых/URL.
- **Тип/контекст ссылок:** доля sitewide/footer (высокая = низкая ценность),
  dofollow-доля, editorial-in-content vs каталоги/списки.
- **Тематическая релевантность доноров:** линкующие сайты в родственной нише
  (для VPN/tech — tech/privacy = золото) или рандом? Прокси естественности → `topical_relevance` (0..1).
- **Гео/TLD доноров** относительно целевого гео (RU).

## Стадия D — Чистота истории (Wayback + индекс)
- **Wayback:** реконструировать, что домен хостил во времени. Флаги прошлого:
  adult / pharma / casino / gambling / warez / спам. Флаг **смены тематики**
  (был пекарней → стал казино = отравлен). Предпочтение — домены, что были
  цельным реальным сайтом.
- **Эхо индексации:** старый контент ещё в индексе Google? (прокси, что не
  деиндексирован/не под баном) → `indexed_echo`.
- **Возраст/непрерывность:** реальный возраст (первый снапшот) vs дата регистрации.

## Стадия E — Риск / санкции
- **РКН реестр:** есть в реестре → заблокирован в РФ → **reject** (`rkn_listed`).
- **Спам-блэклисты:** Spamhaus DBL / SURBL (DNS-based) → флаг/reject (`blacklisted`).
- **Товарный знак** в строке домена → юр-риск (`trademark_risk`).
- **Прокси Google-санкций:** прямой manual-action не проверить; косвенно — обвал
  истории трафика + деиндексация.

## Стадия F — Скор и решение
Композитный `score` = взвешенная сумма (веса в `scoring_config`):
`+` качество RD, чистота анкоров, доля live, тематическая релевантность, чистота истории, возраст
`−` штрафы: блэклист, прошлая плохая категория, РКН, смена тематики.
Пороги → **approve / manual-review / reject**. Человек смотрит approve/manual-review
перед выкупом (плюс сам гейт выкупа). Всё раскладывается в `score_breakdown` (jsonb),
чтобы решение было прозрачным.

## Какие поля модели это заполняют
`domains`: dr, referring_domains, live_referring_domains, backlinks, anchors,
spam_anchor_ratio, topical_relevance, age_years, first_seen, wayback_checked,
prior_flags, indexed_echo, rkn_listed, blacklisted, trademark_risk, clean, score, score_breakdown.
